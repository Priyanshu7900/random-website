SimpleRISC Architecture

Instructions

Basic Characteristics

  • Instruction Set Architecture (ISA) 32-bit
  • Registers 16 registers (r0 - r15)
  • General Purpose Registers r0 - r13
  • Stack Pointer r14 (sp)
  • Return Address Register r15 (ra)
  • Flag Register Used by the cmp (compare) instruction for conditional branches
  • Total Instructions 21
  • Instruction Format Mostly 3-address format (destination first, sources later)
  • Data Instructions Register (R) Type & Immediate (I) Type
  • Max Operands per Instruction 3

Instruction Encoding

The following rules define how instructions are encoded into the 32-bit instruction register:

  • Opcode 5 bits (Most Significant Bits)
  • 6th Bit R/I Type Indicator (0 = Register Type, 1 = Immediate Type)

Modifier Values

15th MSB 16th MSB Meaning
0 0 Default (2's complement)
0 1 Unsigned (u)
1 0 Higher Order (h)

Encoding Formats

0-Address Instruction

Opcode: 5 bits, Remaining 27 bits: Zero

1-Address Instruction

Opcode: 5 bits, 27-bit relative address (Offset * 4 added to PC)

3-Address Instruction

Opcode: 5 bits, R/I Type Indicator: 6th bit

  • R-Type: Rd (4 bits), Rs1 (4 bits), Rs2 (4 bits)
  • I-Type: Rd (4 bits), Rs1 (4 bits), Modifier (2 bits), Immediate (16 bits)

2-Address Instructions

Opcode: 5 bits

  • cmp: Affects flag register, influences branch instructions. R-Type uses Rs1, Rs2. I-Type uses Rs1 and an 18-bit Immediate. Rd is zero.
  • not, mov: R-Type uses Rd and Rs2, with Rs1 set to zero. I-Type uses Rd and an 18-bit Immediate.

Load & Store Instructions

Opcode: 5 bits, Rd (4 bits), Rs1 (4 bits), Immediate (18 bits including modifier)
